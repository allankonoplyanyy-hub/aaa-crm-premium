## 5. Найденные и исправленные ошибки (за время подготовки)

1. `Button render={<Link>}` без `nativeButton={false}` — ошибка гидрации (дашборд).
2. Select без пропа `items` показывал value вместо label (pipeline, диалоги).
3. Горизонтальный overflow из-за отсутствия `min-w-0` на `SidebarInset`.
4. Кнопка «Создать» в диалоге задачи была заблокирована из-за пустого обязательного срока — добавлено значение по умолчанию.
5. Иконка `Instagram` отсутствует в установленной версии lucide-react — заменена.
6. Инбокс на мобильных сжимал тред в полосу — переделан в master-detail.
7. `ToggleGroup` (Base UI) ожидает массив в `value` — исправлен тип.
8. `hooks/use-mobile.ts`: setState внутри effect (ошибка `react-hooks/set-state-in-effect`) — переписан на `useSyncExternalStore`.
9. ESLint 10 несовместим с eslint-plugin-react — зафиксирован ESLint 9.
10. Повреждённые символы (mojibake) в трёх строках сид-данных — исправлены.

## 6. Способ хранения данных (подтверждение)

- **PostgreSQL — НЕ используется.**
- **localStorage — НЕ используется.**
- **In-memory store — используется** (`lib/server/store.ts`, singleton на `globalThis`).
- **После рестарта сервера исчезнут:** все созданные/изменённые сделки, задачи, контакты, сообщения — база пересоздаётся из сида `lib/server/seed.ts`.
- **Перед production заменить:** in-memory store на PostgreSQL-репозиторий, демо-пароль на реальную аутентификацию с хешированием, неподписанную cookie на подписанную сессию. Полный список — CODEX_HANDOFF.md, раздел 11.

## 7. Оставшиеся риски

1. **Эфемерность данных на Vercel**: serverless-инстансы перерабатываются — демо-изменения могут сбрасываться между визитами. Это ожидаемое поведение демо, но его нужно проговаривать при показе.
2. Cookie сессии не подписана: подделка userId возможна. Приемлемо только для демо без реальных данных.
3. Один пароль на все аккаунты.
4. Нет E2E-тестов — регрессии UI ловятся только вручную.
5. `revalidate`/кеширование не настраивалось: все данные запрашиваются динамически, при росте объёма потребуется пагинация.
6. Lint-правило `react/no-unescaped-entities` переведено в warning (русская типографика в копирайтинге) — не влияет на корректность.

## 8. Вердикт

Проект готов к технической проверке Codex как **демо**. Он НЕ является
production-ready: отсутствуют PostgreSQL, настоящая аутентификация и резервное
копирование (см. блокеры в CODEX_HANDOFF.md).
